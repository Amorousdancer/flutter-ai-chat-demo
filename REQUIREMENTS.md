# OfferLab AI 面试与职场沟通陪练 App 需求文档

## 1. 项目背景

当前 AI 对话类应用已经从泛娱乐陪伴扩展到学习、求职、职场成长等场景。相比单纯角色陪聊，OfferLab 选择面试与职场沟通陪练作为切入点，通过 AI 扮演 HR、技术面试官、主管、客户等角色，帮助用户在真实压力场景前进行低成本练习。

本项目用于展示 Flutter 快速开发能力、AI 对话产品设计能力、移动端交互完整性和 MVP 落地能力。

## 2. 目标用户

主要用户：

- 应届生、转行求职者、正在准备面试的人
- 希望提升表达能力的职场新人
- 需要练习薪资谈判、离职沟通、汇报沟通的人

用户痛点：

- 不知道面试官会如何追问
- 回答问题时逻辑不清晰
- 缺少真实对话练习环境
- 不方便找真人反复模拟面试

## 3. 核心功能

### 3.1 场景选择

- Flutter 开发面试
- HR 综合面试
- 压力面试
- 薪资谈判
- 离职沟通
- 客户沟通

### 3.2 AI 对话陪练

- AI 根据所选场景扮演不同角色
- 用户输入回答
- AI 进行追问、反馈或推进对话
- 支持友好模式、压力模式、深挖模式

### 3.3 练习结果反馈

- 给出本轮表现评分
- 分析表达清晰度、逻辑完整度、专业匹配度
- 提供优化后的回答示例
- 标记风险表达，如过度负面、缺少重点、回答太空泛

### 3.4 练习记录

- 展示最近练习场景
- 查看历史对话摘要
- 显示最近得分和建议提升方向

### 3.5 我的页面

- 用户基础信息
- 练习统计
- 关于应用
- 版本信息

## 4. 非功能需求

### 4.1 安全性

- 不展示真实隐私数据
- 用户输入内容仅用于当前练习场景
- 对手机号、身份证、公司机密等敏感信息提示用户谨慎输入

### 4.2 性能

- 页面切换流畅，无明显卡顿
- 首页和消息页首屏加载时间控制在 1 秒内
- 本地模拟数据情况下，聊天消息发送后 0.5 到 1.5 秒内展示 AI 回复效果

### 4.3 可用性

- 核心路径清晰：选择场景、开始对话、查看反馈
- 移动端单手操作友好
- 输入框、按钮、反馈卡片在小屏幕下不遮挡、不溢出
- 无网络或未接入真实 AI 时，可使用本地模拟回复完成演示

### 4.4 可维护性

- 场景、角色、模拟回复使用结构化数据配置
- 页面模块拆分清晰，便于后续接入真实 AI API

## 5. MVP 范围

本期 MVP 只做完整可演示闭环：

- 首页场景列表
- 场景详情页
- AI 模拟聊天页
- 本地预设 AI 回复
- 对话结束后的评分反馈页
- 练习记录页
- 我的页面
- 深色或专业简洁风格 UI

MVP 目标是让 HR 能在 3 分钟内体验到完整流程：选择一个面试场景，进行几轮对话，拿到一份反馈报告。

## 6. 不在本期做的内容

- 用户登录和注册
- 真实支付系统
- 真实大模型 API 接入
- 语音输入和语音播报
- 简历上传和解析
- 多端数据同步
- 社区分享
- 企业版后台
- 复杂权限系统
- 长期学习计划和课程体系

以上功能可作为后续扩展方向，但不进入本期 MVP，避免影响 demo 完成度。

## 7. 验收标准

- 用户可以从首页选择至少 4 个不同练习场景
- 用户可以进入任意场景并完成一轮对话
- 聊天页至少支持用户发送消息、AI 模拟回复、消息滚动
- 对话结束后可以生成一份反馈结果
- 反馈结果包含评分、问题分析、优化建议
- 练习记录页可以展示最近练习内容
- App 至少包含首页、聊天页、反馈页、记录页、我的页
- UI 风格统一，移动端布局无明显错位、遮挡、文字溢出
- 在不接入真实后端的情况下，仍可完整演示核心流程
- 项目可正常运行，并能向 HR 清晰说明后续如何接入真实 AI API
